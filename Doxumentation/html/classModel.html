Detailed Description

Physical model that we want to use.

We're using an abstract base class to form the foundations of the models preparing for additional set ups. Systems will derive from Model and supply their own spectral analyses for cons2prims, prims2cons functions and the flux vectors. Constructors must require access to public Simulation data.

Flux vector.

Generates the values of the net numerical flux of the conserved variables at the cell faces. Method uses the flux vector splitting method along with a lax-friedrichs approximation to determine the flux. dir corresponds to the direction in which we want the flux: 0=x-direction, 1=y-direction.

Parameters
*conspointer to conserved vector work array. Size is Ncons*Nx*Ny*Nz
*primspointer to primitive vector work array. Size is Nprims*Nx*Ny*Nz
*auxpointer to auxilliary vector work array. Size is Naux*Nx*Ny*Nz
*fpointer to flux vector work array. Size is Ncons*Nx*Ny*Nz
dirdirection in which to generate flux vector. (x, y, z) = (0, 1, 2)

Implemented in TwoFluidEMHD, and SRMHD.

Models that can posess a stiff source term and hence (semi-)implicit time integrators will require a source contribution (and cons2prims method) that applies to a single cell. Each of the arguments are only for a single cell, ie, cons points to an (Ncons,) array, etc.
