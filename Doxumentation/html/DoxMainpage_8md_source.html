3 @section intro Multi-Fluid ElectroMagnetoHydroDynamics
4 @par
5 This is METHOD, a relativistic multi-dimensional, multi-fluid ElectroMagnetoHydroDynamic
6 solver built and maintained by [Alex Wright](http://cmg.soton.ac.uk/people/ajw1e16/)
7 under the guidance of [Dr Ian Hawke](https://www.southampton.ac.uk/maths/about/staff/ih3.page).
8 METHOD is being developed as a result of a PhD project to model neutron star systems
9 with multi-fluid models. As a result, ideal and resistive single fluid models exist
10 that are more conventional in astrophysical model, and a relatively new two-fluid
11 model adapted from Amano 2016.

13 @section install Installation
14 @par
15 The easiest way to install METHOD is to clone the github [repository](https://github.com/AlexJamesWright/METHOD).
16 The repository contains all the source files, tests and makefiles that one needs
17 to get going. To start, copy the HTTPS tag on the main page or run
18 
19  git clone https://github.com/AlexJamesWright/METHOD
20 to copy all files to your machine. There are then multiple options, but the safest
21 is to go to the *Project* directory and build all objects first. Do this by running
22 
23  make build
24 This will compile all the source code and build the rootfinders, and also generate
25 the documetation. To check the build has worked, the best option is to run all the
26 tests. From the *Test* directory, running
27 
28  make test
29 will compile all unit tests and, using the GoogleTest framework, execute the
30 available tests. If any tests fail unexpectedly, update the files on you machine
31 by pulling the repository and try again.

33 @section example Example Simulations
34 @par
35 There is currently work on going to build a set of test cases and examples from
36 which an understanding of the underlying code and processes that are implemented
37 can be understood. These examples range from standard Brio Wu shock tube tests
38 to set ups in which there are exact solutions in the form of self similar current
39 sheets. These examples should be the first port of call for anyone trying to use
40 the METHOD code for the first time, of if you are using a particular aspect of the
41 code for the first time and wish to understand what is required to perform a succesfull
42 simulation.
